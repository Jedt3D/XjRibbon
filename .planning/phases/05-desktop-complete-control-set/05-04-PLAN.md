```xojo
  // === Phase 5: Show/hide group (checkboxes + separator + small button) ===
  Var showHide As XjRibbonGroup = viewTab.AddNewGroup("Show/hide")
  Call showHide.AddCheckBox("Item check boxes", "view.checkboxes")
  Call showHide.AddCheckBox("File name extensions", "view.extensions")
  Var hiddenCb As XjRibbonItem = showHide.AddCheckBox("Hidden items", "view.hidden", True)

  showHide.AddSeparator()
  Call showHide.AddSmallButton("Hide selected", "view.hide_selected")
  
  // === Phase 5: Panes group (SplitButton with menu items) ===
  Var panesGroup As XjRibbonGroup = viewTab.AddNewGroup("Panes")
  Var navPane As XjRibbonItem = panesGroup.AddSplitButton("Navigation pane", "view.nav_pane")
  navPane.AddMenuItem("Navigation pane", "view.nav_pane.toggle")
  navPane.AddMenuItem("Expand to open folder", "view.nav_pane.expand")
  navPane.AddMenuItem("Show all folders", "view.nav_pane.allfolders")
  
```

**WHY `Call` for first two checkboxes:** The return value (XjRibbonItem) is not needed
because the checkboxes start unchecked (default) and require no post-creation configuration.
Using `Call` follows the existing project convention (see `Call zoomGroup.AddLargeButton...`
in the Zoom group).

**WHY `Var hiddenCb` with `#Pragma Unused hiddenCb`:** The third checkbox starts checked
(`True` as third argument). We use a named variable to capture the return and immediately
mark it unused with `#Pragma Unused`. This is the Xojo-idiomatic way to avoid a "variable
never used" compiler warning while still exercising the optional parameter. Matches the
pattern described in RESEARCH.md "Demo Window — Current Structure and Insertion Points".

**WHY `showHide.AddSeparator()` (no `Call`):** `AddSeparator()` is a Sub (returns nothing),
so no `Call` wrapper is needed. Xojo does not require `Call` for Subs.

**WHY `Var navPane` captured:** `navPane` IS used immediately after — three `.AddMenuItem`
calls chain off it. The variable is essential, not orphaned.

**WHY insert before Contextual Tabs:** The contextual tabs (`tableDesign`, `picFormat`) are
separate XjRibbonTab objects — they must not be accidentally appended to `viewTab`. Inserting
before the contextual tab section while `viewTab` is still the active local var is the
correct placement.
