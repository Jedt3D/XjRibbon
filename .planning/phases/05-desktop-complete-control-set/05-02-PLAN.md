<objective>
Add three new public factory methods to `XjRibbonGroup`:
1. `AddSplitButton(caption, tag) As XjRibbonItem` — creates dropdown-style item with split body/arrow
2. `AddCheckBox(caption, tag, initialState) As XjRibbonItem` — creates checkbox-style item
3. `AddSeparator()` — appends a separator item (Sub, no return value)

Purpose: All ribbon items are constructed via factory methods on `XjRibbonGroup` to centralize
ItemType assignment and ensure correct flag combinations (e.g., CheckBox always has IsToggle=True).
This plan introduces the public API for the three new control types.

Output: `desktop/XjRibbonGroup.xojo_code` modified with three new `#tag Method` blocks
inserted after the existing `AddDropdownButton` method (currently lines 43-52) and before
the `Caption` property block (currently line 54).

NOTE: This plan can run in parallel with Plan 01 because it touches a different file.
However, it sets `item.IsSplitButton = True` — that property only exists after Plan 01
completes. Plan 03 (Wave 2) is the consumer; Plan 02 itself does not require Plan 01 to
already be merged because `.xojo_code` files are independently editable XML. The Xojo
compiler will resolve symbol references at IDE-open time, so as long as Plans 01 and 02
both finish before Plan 03 runs, Wave 2 will compile cleanly.
</objective>

Insert exactly:

```
	#tag Method, Flags = &h0
		Sub AddSeparator()
		  Var item As New XjRibbonItem
		  item.ItemType = 4
		  mItems.Add(item)
		End Sub
	#tag EndMethod

```

**WHY ItemType = 4:** Matches `kItemTypeSeparator = 4` constant added by Plan 03 (per REQ-511).

**WHY no Caption / Tag set:** A Separator has no visible label and no interaction. Leaving
them as the XjRibbonItem default empty strings ensures (a) `AssignKeyTips` skips it (the
`If item.KeyTip = ""` guard), (b) hit-test-by-bounds returns Nil because Plan 03 will set
its bounds to zero, (c) DrawGroups Case 4 will skip it without rendering.

**WHY Sub not Function:** REQ-515 specifies the public signature `AddSeparator()` with no
return value. Callers don't need a handle on the separator item — it has no properties to
configure post-creation. Using a Sub means callers do NOT need to wrap the call in `Call`.
